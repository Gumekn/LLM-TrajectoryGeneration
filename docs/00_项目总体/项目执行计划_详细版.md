# 危险轨迹生成与评估系统 - 项目执行计划（详细版）

> **项目周期**: 6周
> **技术方案**: 穷举生成 + LLM剪枝
> **核心目标**: 3天完成轨迹生成模块，2周完成轨迹评估模块（初步+迭代），其余时间完成剩余模块
> **已有基础**: 项目框架文档已完成，目录结构已建立

> **重要说明 - 模块组织原则**：
> 模块不必独立成.py文件，可以是函数或类，整合在同一文件中。
> 但每个函数/类必须明确标注：1)所在文件 2)调用位置 3)功能说明
>
> **Week 1实际实现**：
> Week 1的代码已整合到 `core/processor.py`，而非多个独立文件：
>
> - 数据类型 → VehicleTrajectory, RiskAnalysisResult等
> - 数据加载 → WaymoDataLoader类
> - 风险计算 → RiskCalculator, ScenarioRiskAnalyzer类
> - 片段截取 → ScenarioProcessor类 + 工具函数
>
> 详见 `docs/轨迹生成模块设计.md` 和 `core/processor.py`
>
> **LLM意图驱动轨迹变异系统（Week 1扩展）**：
>
> 已实现模块：
> - `core/stage2/llm/config.py` - LLM客户端（UnifiedLLMClient）
> - `core/stage2/llm/prompt_builder.py` - 提示词构建（意图类型 + SYSTEM_PROMPT + TrajectoryPromptBuilder）
> - `core/stage2/intention_generator.py` - 意图生成器（LLMIntentionGenerator + 核心函数）
> - `core/stage2/mutator.py` - 轨迹变异器
> - `core/stage2/storage.py` - 数据持久化
>
> 使用示例：
> ```python
> from core.stage2.intention_generator import run_intention_analysis
> result = run_intention_analysis(fragment)
> ```

### 数据流转设计

```
Waymo原始pkl文件
      ↓
[Step 1] 数据加载模块 → 输出: 场景数据结构
      ↓
[Step 2] 风险计算模块 → 输出: TTC、风险分数、最危险帧
      ↓
[Step 2] 片段截取模块 → 输出: 危险片段（主车+交互车同步轨迹）
      ↓
[Step 3] 特征提取模块 → 输出: 11维特征向量
      ↓
【穷举生成阶段】（3天）
      ↓
[Step 4] 穷举生成模块 → 输出: 候选轨迹列表（含生成参数）
      ↓
[Step 5] 物理验证模块 → 输出: 通过列表 + 剔除列表
      ↓
【轨迹评估阶段】（2周）
      ↓
[Step 6] RAG检索模块 → 输出: 每条候选的相似案例列表
      ↓
[Step 7] LLM评估模块 → 输出: 评估结果（合理/不合理 + 置信度）
      ↓
[Step 8] 人工审核模块 → 输出: 审核记录 + 知识库更新
      ↓
[Step 9] CARLA适配模块 → 输出: JSON/XOSC格式场景文件
```

### Day 1: 数据加载 + 风险计算模块

> **实际实现**：已整合到 `core/processor.py`

#### 上午任务：数据加载模块

**任务1: 实现core/data_loader.py**

- 功能: 加载Waymo pkl格式数据
- 接口设计:
  - `load_scenario(pkl_path: str) -> Dict`: 加载单个场景
  - `get_vehicle_list(scenario_data: Dict) -> List[Dict]`: 获取车辆列表
  - `extract_vehicle_trajectory(scenario_data: Dict, vehicle_id: str) -> Trajectory`: 提取车辆轨迹

### Day 3: 穷举生成模块（核心）

#### 上午任务：穷举生成器

**任务1: 设计参数空间**

- 参数定义:| 参数名         | 范围            | 步长 | 变异公式                          |
  | -------------- | --------------- | ---- | --------------------------------- |
  | accel_offset   | [-4, 4] m/s²   | 0.5  | accel_new = accel_base + offset   |
  | speed_scale    | [0.7, 1.5]      | 0.1  | vel_new = vel_base × scale       |
  | lateral_offset | [-2, 2] m       | 0.3  | y_new = y_base + offset           |
  | time_shift     | [-1, 1] s       | 0.2  | 插值重采样                        |
  | heading_bias   | [-0.3, 0.3] rad | 0.1  | heading_new = heading_base + bias |
- 全参数空间: 17×9×14×11×6 = 141,372种组合

**任务2: 实现core/trajectory_generator.py**

- 接口设计:
  - `generate_candidates(base_fragment, param_space, n_samples) -> List[CandidateTrajectory]`
- 变异策略:
  - 采样: 网格采样/分层采样/自适应采样
  - 变异: 应用5种变异参数
  - 平滑: 关键时间点插值
- 记录每个候选的生成参数
- 输出: core/trajectory_generator.py

#### 下午任务：物理验证模块

**任务3: 实现core/physics_validator.py**

- 硬约束规则:| 规则ID   | 检查项         | 阈值               |
  | -------- | -------------- | ------------------ |
  | PHYS-001 | 最大纵向加速度 | ≤ 6 m/s²         |
  | PHYS-002 | 最大纵向减速度 | ≤ 8 m/s²         |
  | PHYS-003 | 最大横向加速度 | ≤ 4 m/s²         |
  | PHYS-004 | 最大速度       | ≤ 35 m/s          |
  | PHYS-005 | 最小速度       | ≥ 0 m/s           |
  | PHYS-006 | 轨迹连续性     | 无断点             |
  | PHYS-007 | 碰撞检测       | 点距离法，阈值0.3m |
- 接口: `validate_batch(candidates) -> (passed, rejected)`
- 输出: core/physics_validator.py

### Week 2: 评估初版实现

#### Day 6-7: RAG知识库模块

**任务1: 实现rag/embedding_encoder.py**

- 功能: 轨迹特征编码
- 接口: `encode(trajectory) -> np.ndarray` (11维)
- 流程: 运动学特征→空间特征→安全指标→几何特征→L2归一化

**任务2: 实现rag/knowledge_base.py**

- 功能: ChromaDB管理
- 接口:
  - `add_case(case_id, embedding, metadata)`: 添加案例
  - `query_similar(embedding, top_k) -> List[Dict]`: 相似检索
  - `get_case(case_id)`: 获取案例
- 持久化到 data/chroma_db/

**任务3: 实现rag/rag_evaluator.py**

- 功能: RAG评估流程
- 接口: `evaluate_candidates(candidates) -> List[EvaluationResult]`
- 流程:
  1. 特征提取→ChromaDB检索top_k相似案例
  2. 计算相似度统计（max, avg, min）
  3. 分支判断: similarity>0.75标准评估, 0.5-0.75辅助评估, <0.5深度分析

#### Day 8-9: LLM评估模块

**任务4: 实现rag/llm_evaluator.py**

- 功能: LLM合理性评估
- 接口:
  - `evaluate_single(trajectory, similar_cases, context) -> EvaluationResult`
  - `evaluate_batch(candidates) -> List[EvaluationResult]`
- 评估Prompt设计:
  - 输入: 轨迹特征、物理验证结果、相似案例
  - 输出: is_reasonable, confidence, reasoning, danger_level
- 评估策略:| 评估结果            | 置信度≥0.8 | 置信度<0.8 |
  | ------------------- | ----------- | ---------- |
  | is_reasonable=true  | 自动通过    | 需人工审核 |
  | is_reasonable=false | 剔除        | 剔除       |

**任务5: 实现rag/self_expansion.py（LLM自补充）**

- 触发条件: max_similarity < 0.7
- 流程:
  - 置信度≥0.8 → 自动入库（标记llm_auto）
  - 置信度<0.8 → 标记need_human_review
