# 基于大语言模型的自动驾驶危险轨迹生成与评估系统

## 项目概述

本项目旨在构建一套完整的危险轨迹生成与评估系统，通过大语言模型（Large Language Model, LLM）对真实驾驶场景中的交互车辆轨迹进行变异，生成具有现实物理可行性的危险驾驶场景，以支持自动驾驶算法在闭环仿真环境中的安全性测试。

---

## 一、轨迹变异策略：交互车轨迹修改的合理性分析

### 1.1 研究动机

自动驾驶系统的安全性测试需要大量涵盖极端工况的危险场景数据。然而，真实世界中危险驾驶事件具有**稀缺性**与**不可重复性**，导致基于自然驾驶数据的测试难以充分覆盖长尾风险场景。因此，本研究提出基于LLM的轨迹变异方法，在保证物理合理性的前提下，通过修改场景中的交互车辆轨迹，人工构造高风险测试用例。

### 1.2 自车轨迹保持不变的必要性

在双车交互场景中，轨迹修改策略存在两种可行方案：修改自车（Ego Vehicle）轨迹或修改交互车（Adversarial Vehicle）轨迹。经系统性分析，**保持自车轨迹不变、仅修改交互车轨迹**的策略具有以下优势：

| 维度 | 修改自车轨迹 | 修改交互车轨迹 |
|------|-------------|---------------|
| **测试基准一致性** | 破坏原始驾驶行为模式，失去对比基准 | 保持自车自然驾驶特性，可建立安全基线 |
| **场景真实性** | 人为编造轨迹可能导致驾驶行为失真 | 自车保持Waymo数据集中的人类驾驶行为 |
| **测试目标聚焦** | 偏离"应对外部危险"的核心测试目标 | 专门测试自车对突发危险的感知与决策能力 |
| **闭环评估可行性** | 无法量化算法改进效果 | 支持同一危险场景的多轮对比实验 |

### 1.3 技术实现优势

在CARLA仿真环境中，修改交互车轨迹的实现路径更为简洁：
- **自车控制**：可采用轨迹回放（Replay Mode）或接入自动驾驶算法（Algorithm Mode）
- **交互车控制**：通过Python API直接执行LLM生成的轨迹序列
- **数据采集**：支持同步记录自车算法响应、两车相对运动状态及碰撞事件

## 二、闭环仿真测试框架

### 2.1 系统架构

本系统采用三层架构设计：

```
数据生成层 (LLM-2) → 仿真执行层 (CARLA) → 评估分析层
```

#### 2.1.1 数据生成层

该层负责将Waymo原始场景数据转化为可用于闭环测试的危险场景：

1. **场景加载**：从Waymo数据集中提取主车与交互车的全局坐标轨迹
2. **风险计算**：基于相对距离、相对速度及TTC（Time To Collision）指标识别危险片段
3. **轨迹变异**：通过LLM生成交互车的危险轨迹变体
4. **质量验证**：结合物理约束检查与RAG（Retrieval-Augmented Generation）知识库评估生成轨迹的合理性

#### 2.1.2 仿真执行层

基于CARLA仿真引擎构建虚拟测试环境：

| 组件 | 功能描述 | 数据来源 |
|------|---------|---------|
| 自车（Ego） | 执行正常驾驶行为或运行待测AD算法 | Waymo原始轨迹 / 算法决策 |
| 交互车（Adv） | 执行LLM生成的危险轨迹，注入测试风险 | LLM变异生成 |
| 环境仿真 | 提供道路几何、交通标识、光照条件 | CARLA内置地图 |

#### 2.1.3 评估分析层

量化评估自动驾驶系统在危险场景下的表现：

- **安全指标**：最小TTC、碰撞发生率、安全响应时间
- **舒适指标**：加速度变化率（Jerk）、横摆角速度
- **算法性能**：感知延迟、决策时间、轨迹规划合理性

### 2.2 测试流程

系统支持两种测试模式：

**模式一：基线测试（Baseline Test）**
```
原始Waymo场景 → 自车轨迹回放 → 交互车轨迹回放 → 记录正常表现基线
```

**模式二：危险场景测试（Adversarial Test）**
```
LLM变异危险场景 → 自车轨迹回放/算法控制 → 交互车执行危险轨迹 → 对比基线评估算法性能
```

通过对比两种模式下的系统响应差异，可量化评估自动驾驶算法在突发危险工况下的鲁棒性。

## 三、项目结构

```
LLM-2/
├── app.py                      # Streamlit主应用程序入口
├── config.yaml                 # 系统配置参数
├── core/                       # 核心处理模块
│   ├── __init__.py
│   ├── data_loader.py          # Waymo数据加载与解析
│   ├── scene_processor.py      # 场景预处理（风险计算、片段截取）
│   ├── trajectory_generator.py # LLM轨迹变异生成
│   ├── physics_validator.py    # 物理约束验证
│   └── carla_adapter.py        # CARLA数据格式转换
├── rag/                        # RAG知识库模块
│   ├── __init__.py
│   ├── knowledge_base.py       # ChromaDB向量数据库管理
│   ├── embedding_encoder.py    # 轨迹特征向量化
│   ├── rag_evaluator.py        # 检索增强评估
│   └── self_expansion.py       # 知识库自扩充机制
├── simulation/                 # 仿真模块
│   ├── __init__.py
│   ├── simulator_2d.py         # 二维可视化仿真
│   └── replay_engine.py        # 轨迹回放引擎
├── ui/                         # 用户界面组件
│   ├── __init__.py
│   ├── scene_selector.py       # 场景与车辆选择界面
│   ├── generation_panel.py     # 轨迹生成控制面板
│   ├── review_panel.py         # 人工审核界面
│   ├── visualization.py        # 可视化组件
│   └── knowledge_panel.py      # 知识库管理界面
└── data/                       # 数据存储目录
    ├── generated/              # 生成的危险轨迹
    ├── reviewed/               # 审核后的轨迹
    ├── carla_ready/            # CARLA格式输出
    └── chroma_db/              # ChromaDB持久化数据
```

## 四、关键技术选型

| 模块 | 技术方案 | 选型理由 |
|------|---------|---------|
| 向量数据库 | ChromaDB | 本地部署、零配置、Python原生接口 |
| Web界面 | Streamlit | 纯Python实现、快速原型开发、组件丰富 |
| LLM接口 | OpenAI API / Claude API | 支持复杂推理任务、输出结构化数据 |
| 仿真引擎 | CARLA 0.9.15 | 高保真物理仿真、ROS桥接支持、开源免费 |
| 轨迹嵌入 | 统计特征 + Sentence-Transformer | 兼顾轨迹特征与语义描述 |

## 五、数据流与处理流程

```
Waymo原始数据
    ↓
[Step 1] 场景加载与车辆选择
    ↓
[Step 2] 风险计算与危险片段定位
    ↓
[Step 3] 轨迹特征提取与向量化
    ↓
[Step 4] LLM轨迹变异生成
    ↓
[Step 5] 物理约束验证（硬过滤）
    ↓
[Step 6] RAG检索相似案例
    ↓
[Step 7] LLM合理性评估
    ↓
[Step 8] 人工审核与知识库更新
    ↓
[Step 9] CARLA格式导出与闭环测试
```

---

## 六、核心创新点

1. **RAG驱动的轨迹评估**：无需依赖真实碰撞数据，通过检索增强生成技术实现轨迹合理性自动评估
2. **人机协同的知识库构建**：结合LLM自动判断与人工审核，支持知识库的持续学习与扩充
3. **双模式仿真测试**：支持轨迹回放基线测试与算法闭环测试，实现危险场景的可重复验证
4. **物理-语义联合约束**：通过物理规则硬过滤与语义推理软约束，确保生成轨迹的现实可行性
