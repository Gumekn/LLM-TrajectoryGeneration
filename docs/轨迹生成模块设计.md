## 一、模块概述

轨迹生成模块是危险轨迹生成与评估系统的第一阶段，负责从Waymo原始数据中提取危险交互片段，并为后续的穷举生成和LLM评估阶段准备数据。

### 1.1 处理流程

```
Waymo原始pkl文件
      ↓
[Step 1] WaymoDataLoader - 数据加载
         - 加载pkl文件
         - 获取移动车辆列表（排除静止车辆）
         - 用户选择主车（默认sdc_id）
      ↓
[Step 2] ScenarioRiskAnalyzer - 风险分析
         - 计算TTC（纵向/横向）
         - 计算综合风险分数
         - 定位最危险帧（锚点帧）
      ↓
[Step 3] should_skip_fragment - 边界检查
         - 检查锚点帧是否太靠近场景边界
         - 跳过前/后截取时长不足的交互车
      ↓
[Step 4] FragmentExtractor - 片段截取
         - 截取锚点前后指定时长
         - 边界自动处理
         - 主车+交互车同步
      ↓
[Step 5] 特征提取 - 11维特征向量
         - 交互车轨迹特征(5维)
         - 交互特征(6维)
      ↓
[Step 6] JSON持久化 - data/processed/{scenario_id}_{ego_id}.json
         - 无片段时不生成JSON
         - 输出原因分析
```

### 1.2 主车选择流程

1. **加载场景数据**
2. **检索移动车辆**：计算每辆车的平均速度，排除静止车辆（速度<阈值）
3. **显示车辆列表**：显示所有移动车辆的ID、速度、有效帧数，默认选择用 `*` 标记
4. **用户选择**：用户输入主车ID（或使用默认的sdc_id）
5. **后续处理**：所有分析基于用户选择的主车进行

### 1.3 输出产物

每个主车-场景组合生成一个JSON文件，包含：
- 场景元数据
- 所有危险片段列表（每个关键车一个片段）
- 每个片段包含：轨迹数据、交互特征、11维特征向量

## 三、核心算法设计

### 3.1 坐标系转换

Waymo数据中速度和加速度是局部坐标系表示，需要转换为全局坐标系进行计算：

```python
# 局部速度 → 全局速度
gx = vx * cos(heading) - vy * sin(heading)
gy = vx * sin(heading) + vy * cos(heading)

# 全局速度 → 局部速度
lx = gx * cos(heading) + gy * sin(heading)
ly = -gx * sin(heading) + gy * cos(heading)
```

### 3.2 TTC计算

TTC（Time-To-Collision）计算公式：

```python
# 纵向TTC（与Data_Processor.py一致）
tl = np.where((d_l * v_l) < 0, -d_l / v_l, np.inf)
ta = np.where((d_a * v_a) < 0, -d_a / v_a, np.inf)
```

### 3.3 风险分数计算

综合风险分数基于距离评分和TTC评分的加权平均：

| 评分项 | 范围 | 说明 |
|--------|------|------|
| 距离评分 s_d | 0-5 | 0.3m内=0，5m外=5 |
| TTC评分 s_t | 0-5 | 0.15s内=0，5s外=5 |

```python
# 方向权重（判断交互的主要方向）
if 角度在±45°内 or 角度>135°:
    w = 1.0  # 纵向为主
else:
    w = 0.0  # 横向为主

# 综合评分
dsc = s_dl * w + s_da * (1 - w)  # 距离综合分
tsc = s_tl * w + s_ta * (1 - w)  # TTC综合分
risk_score = int((dsc + tsc) / 2)  # 取整
```

### 3.4 危险类型判断

不预设场景类型，而是基于交互特征推断：

```python
def infer_danger_type(ttc_long, ttc_lat, heading_diff):
    min_ttc_long = min(有效ttc_long序列)
    min_ttc_lat = min(有效ttc_lat序列)
    mean_heading_diff = mean(|航向差|)

    # 1. 纵向接近为主
    if min_ttc_long < min_ttc_lat * 0.5 and |mean_heading_diff| < 30°:
        return "rear_end" if heading_diff > 0 else "head_on"

    # 2. 横向接近为主
    if min_ttc_lat < min_ttc_long * 0.5:
        return "cut_in" if |heading_diff| > 45° else "crossing"

    # 3. 混合型
    return "mixed"
```

### 3.5 锚点帧边界检查

```python
def should_skip_fragment(anchor_frame, n_before_frames, n_after_frames, total_frames,
                         min_before_frames=10, min_after_frames=15):
    # 检查锚点帧到开始边界
    if anchor_frame < min_before_frames:
        return True, f"锚点帧{anchor_frame}距开始仅{anchor_frame}帧 < 最小要求{min_before_frames}帧"

    # 检查锚点帧到结束边界
    frames_after_anchor = total_frames - anchor_frame - 1
    if frames_after_anchor < min_after_frames:
        return True, f"锚点帧{anchor_frame}距结束仅{frames_after_anchor}帧 < 最小要求{min_after_frames}帧"

    return False, ""
```

## 四、11维特征设计

### 4.1 特征定义

| 索引 | 特征名 | 计算方式 | 单位 | 说明 |
|------|--------|---------|------|------|
| 0 | mean_speed | mean(\|v\|) | m/s | 交互车平均速度 |
| 1 | max_speed | max(\|v\|) | m/s | 交互车最大速度 |
| 2 | speed_std | std(\|v\|) | m/s | 速度波动程度 |
| 3 | mean_accel | mean(\|a\|) | m/s² | 交互车平均加速度 |
| 4 | max_accel | max(\|a\|) | m/s² | 交互车最大加速度 |
| 5 | min_ttc_long | min(TTC_long) | s | 最小纵向TTC |
| 6 | mean_ttc_long | mean(TTC_long) | s | 平均纵向TTC |
| 7 | min_rel_dist | min(rel_dist) | m | 最小相对距离 |
| 8 | max_closing_speed | max(-v_long) | m/s | 最大接近速度 |
| 9 | trajectory_length | sum(\|ds\|) | m | 行驶距离 |
| 10 | max_curvature | max(\|dθ/ds\|) | 1/m | 最大曲率 |

### 4.2 特征分类

**第一部分：交互车轨迹特征（0-4）**
- 基于交互车自身运动状态计算
- 反映交互车的驾驶行为特征

**第二部分：交互特征（5-10）**
- 基于主车-交互车相对运动计算
- 反映危险交互的严重程度

### 4.3 曲率计算

```python
# 曲率 = 航向角变化 / 弧长
dtheta = diff(heading)
ds = |diff(position)|
curvature = |dtheta| / ds
```

## 五、截取逻辑

### 5.1 截取参数

| 参数 | 默认值 | 命令行参数 | 说明 |
|------|--------|-----------|------|
| n_before_sec | 2.0秒 | --n-before | 锚点前截取时长 |
| n_after_sec | 3.0秒 | --n-after | 锚点后截取时长 |
| min_before_frames | 10帧 | --min-before-frames | 锚点前最小帧数 |
| min_after_frames | 15帧 | --min-after-frames | 锚点后最小帧数 |
| sampling_rate | 10Hz | - | 采样率 |

### 5.2 边界处理

```python
def extract_fragment_slice(anchor_frame, n_before_frames, n_after_frames, total_frames):
    # 计算理想边界
    start = anchor_frame - n_before_frames
    end = anchor_frame + n_after_frames + 1

    # 左边界处理
    if start < 0:
        end -= start
        start = 0

    # 右边界处理
    if end > total_frames:
        start -= (end - total_frames)
        start = max(0, start)
        end = total_frames

    return start, end
```

## 六、使用方法

### 6.1 主程序运行（推荐）

```bash
python main.py
```

### 6.2 命令行参数

```bash
python main.py [选项]
  --scenario, -s          场景ID (默认: 10135f16cd538e19)
  --ego-id, -e             主车ID (默认: 交互选择，优先sdc_id)
  --data-dir, -d           数据目录 (默认: data/waymo-open)
  --output-dir, -o          输出目录 (默认: data/processed)
  --n-before               锚点前时长(秒) (默认: 2.0)
  --n-after                锚点后时长(秒) (默认: 3.0)
  --risk-threshold, -r     风险阈值 (默认: 3)
  --speed-threshold        速度阈值(m/s) (默认: 0.5)
  --min-before-frames      锚点前最小帧数 (默认: 10)
  --min-after-frames       锚点后最小帧数 (默认: 15)
```

### 6.3 基本使用（代码方式）

```python
from core import WaymoDataLoader, ScenarioProcessor

# 初始化处理器
processor = ScenarioProcessor(
    data_dir="data/waymo-open",
    output_dir="data/processed"
)

# 处理单个场景
scenario = processor.process_scenario(
    scenario_id="10135f16cd538e19",
    ego_id="1811",
    n_before_sec=2.0,
    n_after_sec=3.0,
    risk_threshold=3,
    min_before_frames=10,
    min_after_frames=15
)
```
