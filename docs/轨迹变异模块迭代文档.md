## 一、模块概述

轨迹变异模块基于意图分析结果，对交互车轨迹进行参数化变异穷举，生成多样化的危险轨迹变体，供后续仿真测试使用。

### 1.1 与意图生成模块的关系

```
Stage 1 输出 (data/processed/)
        │
        ▼
Stage 2a: 意图生成 (run_intention.py)
        │ intention_frames[]
        ▼
Stage 2b: 轨迹变异 (run_mutation.py)
        │
        ▼
Stage 2 输出 (data/variants/)
```

- **意图生成** (`intention_generator.py`)：输入片段 → LLM调用 → 输出意图帧序列
- **轨迹变异** (`mutator.py`)：输入带意图片段 → DFS穷举 → 输出变异轨迹列表

## 二、核心算法

### 2.1 算法流程

```
意图帧序列 (intention_frames)
        │
        ▼  merge_intentions() 意图合并（游程编码）
意图区块列表 (IntentBlock[])
        │
        ▼  DFS 遍历每个 Block
对每个 Block:
  1. 获取意图变异配置 (INTENTION_MUTATIONS)
  2. 穷举 (delta_a_mag, delta_omega) 组合
  3. 运动学积分递推该 Block 内所有帧
  4. 计算 Block 末帧风险分数
  5. 非末 Block → Top-K% 剪枝
  6. 递归下一 Block
        │
        ▼  叶子节点: compute_risk_score() 计算最终风险分数
变异轨迹列表 (variants[])
```

### 2.2 意图合并（游程编码）

相邻且意图相同的帧合并为一个 `IntentBlock`，减少变异次数：

```
帧0  cruise_maintain ─┐
帧1  cruise_maintain ─┤
帧2  decelerate ──────┼── Block(cruise_maintain, 0-2)
帧3  decelerate ──────┤
帧4  accelerate ──────┼── Block(decelerate, 2-4)
帧5  accelerate ──────┤
...                   ▼
                   Block(accelerate, 4-51)
```

### 2.3 变异参数

| 意图 | delta_a_mag (m/s²) | delta_omega (rad/s) | 组合数 |
|------|---------------------|---------------------|--------|
| cruise_maintain | 扰动 N(0,0.05) | 扰动 N(0,0.004) | ~1 |
| accelerate_through | 0.2, 0.4, 0.6 | 0 | 3 |
| decelerate_to_yield | -0.3, -0.6, -1.0 | 0 | 3 |
| decelerate_to_stop | -0.3, -0.6, -1.0 | 0 | 3 |
| emergency_brake | -3.0, -5.0, -8.0 | 0 | 3 |
| lane_change_left | -0.2, 0, 0.2 | 0.01, 0.02, 0.03 | 9 |
| lane_change_right | -0.2, 0, 0.2 | -0.03, -0.02, -0.01 | 9 |
| turn_left | -0.2, 0, 0.2 | 0.02, 0.04, 0.06 | 9 |
| turn_right | -0.2, 0, 0.2 | -0.06, -0.04, -0.02 | 9 |
| go_straight | -0.2, 0, 0.2 | 0 | 3 |

### 2.4 物理常数

| 常数 | 值 | 说明 |
|------|-----|------|
| DT | 0.1s | 时间步长（10Hz采样率） |
| A_MAG_MAX | 10.0 m/s² | 合加速度上限 |
| OMEGA_MIN/MAX | ±0.35 rad/s | 横摆角速度边界 |
| SIGMA_A | 0.05 m/s² | 加速度扰动标准差 |
| SIGMA_OMEGA | 0.004 rad/s | 角速度扰动标准差 |

## 三、文件结构

```
core/stage2/
├── intention_generator.py    # 意图生成（LLMIntentionGenerator + 核心函数）
├── mutator.py               # 轨迹变异（核心算法）
├── storage.py              # 数据持久化
├── run_intention.py         # 意图生成入口
└── run_mutation.py          # 轨迹变异入口
```

### 3.1 mutator.py 核心函数

| 函数 | 职责 |
|------|------|
| `merge_intentions()` | 意图帧序列 → IntentBlock 列表 |
| `extract_initial_state()` | 轨迹 → 初始运动学状态 |
| `mutate_vector_acceleration()` | 矢量加速度变异 |
| `kinematically_integrate()` | 运动学积分一步 |
| `get_mutation_combinations()` | 穷举变异参数组合 |
| `compute_block_risk_score()` | 逐 Block 简化风险分数 |
| `compute_risk_score()` | 最终风险分数（复用 RiskCalculator） |
| `dfs_mutate_with_pruning()` | DFS + 逐 Block Top-K% 剪枝 |
| `build_variant_output()` | 变异状态 → 输出格式 |

### 3.2 数据类

| 类 | 字段 |
|----|------|
| `IntentBlock` | intent, start_frame, end_frame, duration |
| `TrajectoryState` | v, a_x, a_y, theta, omega, x, y |
| `IntentionMutation` | delta_a_mag_values[], delta_omega_values[], is_perturbation |

## 四、使用方式

### 4.1 入口脚本（推荐）

```bash
# 意图生成
python -m core.stage2.run_intention -i data/processed/10135f16cd538e19_1811.json

# 轨迹变异
python -m core.stage2.run_mutation -i data/intention/frag_xxx_with_intention.json
```

### 4.2 代码调用

```python
from core.stage2.mutator import TrajectoryMutator

mutator = TrajectoryMutator(random_seed=42)
variants = mutator.mutate(fragment_with_intention, top_k=10)
```

### 4.3 输入数据格式

带意图片段 JSON（Stage 2a 输出）：

```json
{
  "fragment_id": "frag_10135f16cd538e19_1811_1707_87",
  "metadata": { "scenario_id": "...", "anchor_frame": 87, ... },
  "ego_trajectory": { "positions": [...], "headings": [...], ... },
  "target_trajectory": { "positions": [...], "headings": [...], ... },
  "intention_analysis": {
    "intention_frames": [
      { "frame": 0, "frame_type": "anchor", "intention": "cruise_maintain", "reasoning": "保持" },
      { "frame": 25, "intention": "decelerate_to_yield", "reasoning": "减速让行" }
    ]
  }
}
```

### 4.4 输出数据格式

变异结果 JSON（Stage 2b 输出）：

```json
{
  "fragment_id": "frag_10135f16cd538e19_1811_1707_87",
  "metadata": { ... },
  "variant_count": 3,
  "variants": [
    {
      "variant_id": 0,
      "positions": [[x, y, z], ...],
      "headings": [...],
      "velocities": [[v, 0], ...],
      "accelerations": [[ax, ay], ...],
      "valid": [true, ...]
    },
    ...
  ]
}
```

## 五、与旧版设计的差异

### 5.1 文件路径变更

| 旧版 | 当前版 |
|------|--------|
| `core/trajectory_mutator.py` | `core/stage2/mutator.py` |
| `core/llm_intention_generator.py` | `core/stage2/intention_generator.py` |
| `core/llm/intention_models.py` | 已删除（合并到 intention_generator.py） |

### 5.2 关键改进

1. **意图生成与变异解耦**：意图生成专注 LLM 调用和解析，变异专注穷举算法
2. **入口脚本化**：`run_intention.py` / `run_mutation.py` 提供标准化入口
3. **fragments 数组支持**：`run_intention.py` 支持处理包含多个片段的 JSON 文件
4. **无循环导入**：修复了 `intention_models.py` 删除后的循环导入问题

### 5.3 未变化部分

- 意图类型定义（10种）：`DrivingIntention` 仍在 `mutator.py`
- 变异参数（delta_a_mag / delta_omega）：与设计规范一致
- DFS + Top-K% 剪枝算法：与设计规范一致
- 物理常数：与设计规范一致

---

## 六、已知限制

1. **轨迹时长不定**：当前实现支持任意时长，但未做分段处理
2. **风险分数简化版**：`compute_block_risk_score` 为简化计算，不完全等于 RiskCalculator
3. **末 Block 不剪枝**：Top-K% 剪枝仅在非末 Block 执行
