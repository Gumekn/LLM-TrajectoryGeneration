# 意图驱动轨迹变异穷举方法规范（v2）

## 1. 概述

本规范定义基于驾驶意图的轨迹变异穷举方法。给定一条原始交互轨迹（5s / 50帧 / $\Delta t = 0.1s$），通过意图合并降维、宏动作变异、DFS深度优先搜索和Top-K%危险剪枝，生成所有有效的轨迹变体。

**核心思想**：将高频逐帧变异改为拉长周期的段落式决策（宏动作），在意图合并后的Block边界进行变异，大幅削减计算量至指数级爆炸降维到普通算力可承受的范围。

### 1.1 数量级爆炸解决方案

| 方案 | 原理 | 效果 |
|------|------|------|
| **意图合并（游程编码）** | 相邻相同意图合并为Intent Block | 8~13个意图帧 → 3~5个Block |
| **钦定变异子集** | 每种意图只保留3种最优变异组合 | 每Block仅3种分支 |
| **DFS + Top-K%剪枝** | 生成后按危险分数筛选 | 只保留最危险轨迹 |
| **宏动作递推** | Block内保持$a$和$\omega$恒定 | 避免高频控制抖动 |

## 2. 状态向量定义

### 2.1 运动学状态向量

原始轨迹的每帧数据映射为六维状态向量：

$$S_t = [v_t,\ a_t,\ \theta_t,\ \omega_t,\ x_t,\ y_t]$$

| 符号 | 含义 | 单位 |
|------|------|------|
| $v_t$ | 速度标量 | m/s |
| $a_t$ | 纵向加速度 | m/s^2 |
| $\theta_t$ | 航向角 (heading) | rad |
| $\omega_t$ | 横摆角速度 (yaw rate) | rad/s |
| $x_t$ | X轴坐标 | m |
| $y_t$ | Y轴坐标 | m |

### 2.2 初始状态获取

假设原始轨迹存储于 VehicleTrajectory 结构，时间步长 $\Delta t = 0.1s$：

$$v_0 = |velocities[0]|$$

$$a_0 = accelerations[0][0]$$

$$\theta_0 = headings[0]$$

$$\omega_0 = \frac{headings[1] - headings[0]}{\Delta t}$$

$$x_0 = positions[0][0]$$

$$y_0 = positions[0][1]$$

## 3. 意图合并（游程编码）

### 3.1 问题背景

5秒轨迹中存在8~13个意图帧（如每0.5s一个意图帧，穿插最小TTC、最短车间距等）。若每个意图帧都作为独立变异段，将产生$3^{8-13}$量级的计算爆炸。

### 3.2 游程编码合并算法

**输入**：意图序列，例如
```
[(0.0s, 'cruise'), (0.5s, 'cruise'), (0.8s, 'turn_left'), (1.5s, 'turn_left'), ...]
```

**合并逻辑**：相邻且标签相同的意图合并为一个**意图区块 (Intent Block)**

**输出**：
```
[Block(intent='cruise', start=0.0s, end=0.8s, duration=0.8s),
 Block(intent='turn_left', start=0.8s, end=1.5s, duration=0.7s),
 ...]
```

### 3.3 变异触发时机

**只在每个Block的起始帧进行一次变异**，选定 $(\Delta a, \Delta \omega)$ 后：
1. 更新基础控制量 $a_{block}$ 和 $\omega_{block}$
2. 在该Block持续时间内保持 $a_{block}$ 和 $\omega_{block}$ **恒定**
3. 按0.1s步长进行物理积分递推

## 4. 变异变量与意图剪枝

### 4.1 控制变量边界

| 变量 | 最小值 | 最大值 | 单位 |
|------|--------|--------|------|
| 纵向加速度 $a$ | -8.0 | 3.0 | m/s^2 |
| 横摆角速度 $\omega$ | -0.35 | 0.35 | rad/s |

### 4.2 变异量定义

**Delta_a（纵向加速度增量）**：改变油门/刹车

**Delta_omega（横摆角速度增量）**：改变方向盘打角

注：$\Delta a$ 和 $\Delta \omega$ 代表的是**相对于当前状态的增量**，加负值意味着将当前属性归零（收敛）。

### 4.3 意图钦定变异子集（最大3种/Block）

为每种意图直接钦定符合逻辑的最优变异子集，最大单次分支仅为**3**。

| 意图 | 变异组合 $(\Delta a, \Delta \omega)$ | 语义 |
|------|--------------------------------------|------|
| **cruise_maintain** | (-0.1, 0), (0, 0), (0.1, 0) | 匀速跟车，微调 |
| **accelerate_through** | (0, 0.0087), (0.1, 0), (0.2, 0) | 加速通过 |
| **decelerate_to_yield** | (-0.1, 0), (-0.2, 0), (-0.5, 0) | 减速让行 |
| **decelerate_to_stop** | (-0.1, 0), (-0.2, 0), (-0.5, 0) + 停车特判 | 减速停车 |
| **emergency_brake** | (-0.5, 0), (-1.0, 0), (-2.0, 0) | 紧急制动（锁死方向盘） |
| **lane_change_left** | (0, 0.0087), (0, 0.0175), (0, 0.0349) | 向左变道 |
| **lane_change_right** | (0, -0.0087), (0, -0.0175), (0, -0.0349) | 向右变道 |
| **turn_left** | (-0.1, 0.0175), (-0.2, 0.0175), (0, 0.0349) | 左转 |
| **turn_right** | (-0.1, -0.0175), (-0.2, -0.0175), (0, -0.0349) | 右转 |
| **go_straight** | (-0.1, 0), (0, 0), (0.1, 0) | 直行 |

### 4.4 特殊意图处理

**decelerate_to_stop 特判规则**：
- 若 $v_{t-1} = 0$，强制令 $a_t = 0$，$v_t = 0$
- 保证停车后不再移动

**emergency_brake 特判规则**：
- $\Delta\omega = 0$（锁死方向盘，防止侧翻）
- 强制使用强负向加速度

## 5. 宏动作递推公式（Block内恒定控制）

### 5.1 区块恒定控制量计算

在Block起始帧，根据变异量更新控制量：

$$a_{block} = \max(-8.0,\ \min(3.0,\ a_{t-1} + \Delta a))$$

$$\omega_{block} = \max(-0.35,\ \min(0.35,\ \omega_{t-1} + \Delta \omega))$$

### 5.2 微步长递推公式（每帧执行）

在Block内的每一帧 $i \in [1, N]$，执行0.1s的物理递推：

**速度更新**：
$$v_t = \max(0,\ v_{t-1} + a_{block} \cdot 0.1)$$

**航向角更新**：
$$\theta_t = \theta_{t-1} + \omega_{block} \cdot 0.1$$

**位置更新（运动学积分）**：
$$x_t = x_{t-1} + v_t \cdot \cos(\theta_t) \cdot 0.1$$

$$y_t = y_{t-1} + v_t \cdot \sin(\theta_t) \cdot 0.1$$

### 5.3 VehicleTrajectory 数据结构映射

| 字段 | 映射 |
|------|------|
| `positions[t]` | $[x_t,\ y_t,\ z_{t-1}]$，Z轴高度保持不变 |
| `headings[t]` | $\theta_t$ |
| `velocities[t]` | $[v_t,\ 0]$，局部坐标系：纵向速度 $v_t$，横向速度 0 |
| `accelerations[t]` | $[a_{block},\ v_t \cdot \omega_{block}]$，局部坐标系：纵向加速度，向心加速度 |
| `valid[t]` | True |

## 6. DFS + Top-K% 危险剪枝算法

### 6.1 问题背景

即使有意图合并，3~5个Block仍可能产生 $3^5 = 243$ 条轨迹。需要筛选出最危险（最有研究价值）的轨迹。

### 6.2 危险分数计算

每生成一条轨迹后，计算其危险分数：

$$Risk = \frac{1}{N} \sum_{t=0}^{N-1} \left( \frac{1}{TTC_t} + \frac{1}{Dist_t} \right)$$

其中：
- $TTC_t$：第 $t$ 帧的碰撞时间（Time-To-Collision），越小越危险
- $Dist_t$：第 $t$ 帧的相对距离，越小越危险

### 6.3 Top-K% 剪枝策略

1. **DFS遍历**：使用深度优先搜索遍历所有Block的变异组合
2. **生成分数**：每到达叶子节点（完整轨迹），计算危险分数
3. **Top-K保留**：对所有轨迹按危险分数降序排序，保留前K%（如10%）

### 6.4 伪代码

```
输入: IntentBlocks, K (保留比例)
输出: Top-K% 危险轨迹列表

def dfs(block_idx, current_state, trajectory):
    if block_idx == len(IntentBlocks):
        # 到达叶子节点，计算危险分数
        risk_score = compute_risk(trajectory)
        results.append((trajectory, risk_score))
        return

    block = IntentBlocks[block_idx]
    for (delta_a, delta_omega) in block.mutations:
        # 计算区块恒定控制量
        a_block = clamp(current_state.a + delta_a, A_MIN, A_MAX)
        omega_block = clamp(current_state.omega + delta_omega, OMEGA_MIN, OMEGA_MAX)

        # 递推该Block内所有帧
        for t in range(block.duration_frames):
            new_state = kinematically_integrate(current_state, a_block, omega_block)
            trajectory.append(new_state)
            current_state = new_state

        # 递归处理下一Block
        dfs(block_idx + 1, current_state, trajectory)

# 主流程
all_trajectories = []
dfs(0, initial_state, [])

# Top-K% 筛选
sorted_trajectories = sorted(all_trajectories, key=lambda x: x.risk_score, reverse=True)
top_k_percent = sorted_trajectories[:int(len(sorted_trajectories) * K / 100)]
```

## 7. 完整算法流程

### 7.1 算法总流程

```
输入: TrajectoryFragment (原始轨迹 + 意图帧序列)
输出: Top-K% 危险轨迹变体列表

┌─────────────────────────────────────────┐
│ Step 1: 意图合并（游程编码）              │
│   输入: 8~13个意图帧                     │
│   输出: 3~5个IntentBlock                 │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│ Step 2: 区块变异枚举（DFS）               │
│   每个Block钦定3种变异组合                │
│   3~5个Block → 最多3^5=243条轨迹         │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│ Step 3: 宏动作递推生成轨迹               │
│   对每个Block：                          │
│   - 计算a_block, omega_block            │
│   - 恒定控制量递推该Block内所有帧        │
└─────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│ Step 4: Top-K% 危险剪枝                 │
│   按危险分数排序，保留前K%              │
│   如K=10 → 保留约24条轨迹               │
└─────────────────────────────────────────┘
```

### 7.2 复杂度对比

| 方案 | 意图帧数 | 每帧变异数 | 总变体数 |
|------|----------|------------|----------|
| **旧方案（逐帧穷举）** | 50 | 3~5 | $3^{50}$ ≈ $10^{24}$ (不可行) |
| **新方案（意图合并+DFS）** | 3~5 Block | 3/Block | $3^5$ = 243 |
| **新方案+Top-K%** | 3~5 Block | 3/Block | $243 \times 10\% ≈ 24$ |

## 11. 参考文献

本文档定义的方法适用于自动驾驶轨迹生成与仿真场景，基于以下假设：

- 非完整约束（车辆无侧滑）
- 运动学模型（忽略车辆动力学）
- 时间步长固定为 0.1s（10Hz采样率）
- 意图合并使用游程编码（Run-Length Encoding）
- 宏动作变异：Block内保持控制量恒定
