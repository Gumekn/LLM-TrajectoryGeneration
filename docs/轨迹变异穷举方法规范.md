# 意图驱动轨迹变异穷举方法规范（v3）

## 1. 概述

本规范定义基于驾驶意图的轨迹变异穷举方法。给定一条原始交互轨迹（5s / 50帧 / $\Delta t = 0.1s$），通过意图合并降维、宏动作变异、DFS深度优先搜索和Top-K%危险剪枝，生成所有有效的轨迹变体。

## 核心目标

修改时序轨迹数据中交互车的加速度、航向角两个变量，使安全场景转为碰撞场景。

## 依据

根据车辆物理限制确定两个变量变异的范围，明确颗粒度，在范围内设置N个值，在穷举时从这N个值中选择一个作为变异量，利用递推公式和物理数学公式计算车辆速度位置等变量。

在整条轨迹中，存在某些帧是意图帧，意图定义在prompt_builder.py中有，共计10种意图，意图的作用是引导变量变异方向从而减少变量变异时的可能性数量，缓解指数爆炸问题。请你根据每个意图对应设定每个变量的变异范围，针对每种意图，每个变量有1-4种变异选择可能。

## 变量解释

数据读取意图生成环节之后生成的数据，其中存在主车和交互车轨迹，只修改交互车轨迹。加速度为二维数组，（ax，ay），ax指的是车头朝向的加速度，ay指的是垂直车头方向的加速度，航向角heading单位为弧度。在变异加速度时，采用ax和ay构成的矢量加速度，但是在生成的轨迹中要还原到ax和ay，即变异时由分量转化为合量进行变异，但是变异完转化为分量进行存储和递推；航向角按照弧度计算。

## 解决指数爆炸

1. 意图合并，5s数据中存在多个意图帧，但两帧之间的意图可能是相同的，那么把他们进行合并，从而减少变异次数，直接减少量级。
2. 削减每个变量每次变异的可能性，在意图的基础上，继续限制。比如急刹车那就加速度强制设为0.5/s2，这样减少了每帧变异的数量。
3. 每帧剪枝策略，在轨迹变异过程中加入危险分数评判策略，只取top-K%危险轨迹，从而筛选出最有效的变异枝。危险分数评判策略采用

**核心思想**：

- 将高频逐帧变异改为拉长周期的段落式决策（宏动作）
- 加速度变异针对**矢量加速度**（ax, ay合并为标量+方向）
- 保持状态用**随机扰动**而非固定离散值
- 危险分数复用 `processor.py` 的 `RiskCalculator`

**注意：**

1，核心函数、类写道@/core/llm/intention_models.py 中，轨迹变异代码只起调用、数据加载更新存储、参数设置的功能。

2，意图帧数量不定，根据前一步生成的带意图的轨迹数据决定，因此在合并意图块时需要注意。

3，轨迹时长不定，取决于数据时间，不一定是5s。

4，采用深度优先算法进行穷举，每一个意图块下的连续多帧数据递推生成完之后，按照风险分数进行剪枝，然后前top-k%继续在下一个意图帧时变异穷举。也就是说top-k剪枝只发生在每一个意图块的最后一帧。

### 1.1 数量级爆炸解决方案

| 方案                           | 原理                           | 效果                        |
| ------------------------------ | ------------------------------ | --------------------------- |
| **意图合并（游程编码）** | 相邻相同意图合并为Intent Block | 8~13个意图帧 → 3~5个Block |
| **加速度矢量变异**       | ax, ay合并为矢量，统一变异     | 简化二维变异为一维+方向     |
| **随机扰动保持**         | 保持状态时施加小幅随机扰动     | 更真实的驾驶行为            |
| **DFS + Top-K%剪枝**     | 生成后按危险分数筛选           | 只保留最危险轨迹            |

## 2. 状态向量定义

### 2.1 运动学状态向量

原始轨迹的每帧数据映射为六维状态向量：

$$
S_t = [v_t,\ a_t,\ \theta_t,\ \omega_t,\ x_t,\ y_t]
$$

| 符号         | 含义                  | 单位  |
| ------------ | --------------------- | ----- |
| $v_t$      | 速度标量              | m/s   |
| $a_t$      | 纵向加速度（标量）    | m/s^2 |
| $\theta_t$ | 航向角 (heading)      | rad   |
| $\omega_t$ | 横摆角速度 (yaw rate) | rad/s |
| $x_t$      | X轴坐标               | m     |
| $y_t$      | Y轴坐标               | m     |

### 2.2 矢量加速度

**原始加速度**：$[a_x,\ a_y]$（二维矢量）

**矢量合并**：

$$
a_{mag} = \sqrt{a_x^2 + a_y^2}
$$

$$
a_{angle} = \text{atan2}(a_y,\ a_x)
$$

**局部坐标系映射**：

- $a_x$：纵向加速度（正=加速，负=减速）
- $a_y$：横向加速度（正=左，负=右，向心加速度）

### 2.3 初始状态获取

假设原始轨迹存储于 VehicleTrajectory 结构，时间步长 $\Delta t = 0.1s$：

$$
v_0 = |velocities[0]|
$$

$$
a_{x,0} = accelerations[0][0]
$$

$$
a_{y,0} = accelerations[0][1]
$$

$$
a_{mag,0} = \sqrt{a_{x,0}^2 + a_{y,0}^2}
$$

$$
\theta_0 = headings[0]
$$

$$
\omega_0 = \frac{headings[1] - headings[0]}{\Delta t}
$$

$$
x_0 = positions[0][0]
$$

$$
y_0 = positions[0][1]
$$

## 3. 意图合并（游程编码）

### 3.1 问题背景

5秒轨迹中存在8~13个意图帧。若每个意图帧都作为独立变异段，将产生计算爆炸。

### 3.2 游程编码合并算法

**输入**：意图序列

**合并逻辑**：相邻且标签相同的意图合并为一个**意图区块 (Intent Block)**

**输出**：

```
[Block(intent='cruise', start=0.0s, end=0.8s),
 Block(intent='turn_left', start=0.8s, end=1.5s),
 ...]
```

### 3.3 变异触发时机

**只在每个Block的起始帧进行一次变异**：

1. 在Block起始帧根据意图选定变异参数
2. 在该Block持续时间内保持参数恒定（宏动作）
3. 按0.1s步长进行物理积分递推

## 4. 变异变量定义

### 4.1 控制变量边界

| 变量                 | 最小值 | 最大值 | 单位  |
| -------------------- | ------ | ------ | ----- |
| 合加速度$a_{mag}$  | 0      | 10.0   | m/s² |
| 横摆角速度$\omega$ | -0.35  | 0.35   | rad/s |
| 纵向加速度$a_x$    | -8.0   | 3.0    | m/s² |
| 横向加速度$a_y$    | -5.0   | 5.0    | m/s² |

### 4.2 离散采样步长

| 变量                | 步长            | 说明           |
| ------------------- | --------------- | -------------- |
| 合加速度$a_{mag}$ | 每意图3个固定值 | 直接选择不采样 |
| 航向角$\omega$    | 每意图3个固定值 | 直接选择不采样 |

### 4.3 变异量定义

**合加速度增量（Delta_a_mag）**：

- 针对合加速度 $a_{mag}$ 的增量
- 单位：m/s²
- 含义：改变当前加速度的大小
- 采样方式：每意图3个固定值（缓慢/适中/激进）

**航向角增量（Delta_omega）**：

- 针对横摆角速度 $\omega$ 的增量
- 单位：rad/s
- 含义：改变转向角速度
- 采样方式：每意图3个固定值（缓慢/适中/激进）

### 4.4 随机扰动规则

当意图要求"保持当前状态"时，使用**随机扰动**而非固定值：

$$
Delta\_a \sim N(0,\ \sigma_a)
$$

$$
Delta\_omega \sim N(0,\ \sigma_\omega)
$$

| 扰动参数                          | 值                         |
| --------------------------------- | -------------------------- |
| $\sigma_a$（加速度标准差）      | 0.05 m/s^2                 |
| $\sigma_\omega$（角速度标准差） | 0.004 rad/s (约0.23 deg/s) |

## 5. 意图变异规则（合加速度+航向角，各3选）

### 5.1 变异说明

**合加速度变异**：

- 原始加速度 $[a_x, a_y]$ 合并为合加速度 $a_{mag} = \sqrt{a_x^2 + a_y^2}$
- 变异时直接对合加速度 $a_{mag}$ 进行增量变异
- 变异后还原分量：$a_x = a_{mag,new} \cdot \cos(\theta)$，$a_y = a_{mag,new} \cdot \sin(\theta)$

**每种意图的变异组合**：3（加速度选择）× 3（航向角选择）= 9种

## 6. 宏动作递推公式

### 6.1 合加速度变异流程

1. **计算当前合加速度**：

$$
a_{mag} = \sqrt{a_x^2 + a_y^2}
$$

2. **应用增量变异**：

$$
a_{mag,new} = \text{clamp}(a_{mag} + \Delta a_{mag},\ a_{min},\ a_{max})
$$

3. **还原分加速度**（保持原方向）：

$$
a_x = a_{mag,new} \cdot \cos(\theta)
$$

$$
a_y = a_{mag,new} \cdot \sin(\theta)
$$

### 6.2 区块恒定控制量计算

在Block起始帧，根据变异方案计算控制量：

**合加速度更新**：

$$
a_{mag,block} = \text{clamp}(a_{mag,prev} + \Delta a_{mag},\ a_{min},\ a_{max})
$$

**横摆角速度更新**：

$$
\omega_{block} = \text{clamp}(\omega_{prev} + \Delta \omega,\ \omega_{min},\ \omega_{max})
$$

### 6.3 微步长递推公式

在Block内的每一帧，执行0.1s的物理递推：

**分加速度还原**：

$$
a_x = a_{mag,block} \cdot \cos(\theta)
$$

$$
a_y = a_{mag,block} \cdot \sin(\theta)
$$

**纵向速度更新（局部坐标系）**：

$$
v_{long} = \max(0,\ v_{t-1} + a_x \cdot 0.1)
$$

**全局速度计算（纵向 + 横向贡献）**：

$$
v_{x,global} = v_{long} \cdot \cos(\theta) - a_y \cdot \sin(\theta) \cdot 0.1
$$

$$
v_{y,global} = v_{long} \cdot \sin(\theta) + a_y \cdot \cos(\theta) \cdot 0.1
$$

**位置更新（全局坐标系）**：

$$
x_t = x_{t-1} + v_{x,global} \cdot 0.1
$$

$$
y_t = y_{t-1} + v_{y,global} \cdot 0.1
$$

**航向角更新**：

$$
\theta_t = \theta_{t-1} + \omega_{block} \cdot 0.1
$$

### 6.3 VehicleTrajectory 数据结构映射

| 字段                 | 映射                                       |
| -------------------- | ------------------------------------------ |
| `positions[t]`     | $[x_t,\ y_t,\ z_{t-1}]$                  |
| `headings[t]`      | $\theta_t$                               |
| `velocities[t]`    | $[v_t,\ 0]$                              |
| `accelerations[t]` | $[a_{block},\ v_t \cdot \omega_{block}]$ |
| `valid[t]`         | True                                       |

## 7. 危险分数计算（复用processor.py）

### 7.1 RiskCalculator复用

使用与 `processor.py` 相同的 `RiskCalculator` 计算危险分数：

```python
from core.processor import RiskCalculator, VehicleTrajectory

# 构建轨迹对象
ego_traj = VehicleTrajectory(...)
target_traj = VehicleTrajectory(...)

# 计算风险
calculator = RiskCalculator(ego_trajectory, target_trajectory)
calculator.compute_relative_metrics()
calculator.compute_risk_scores()

# 获取危险分数
risk_scores = calculator.risk_scores  # (N, 7)
max_risk = np.min(risk_scores[:, 6])  # 取第7列（综合评分）最小值
```

### 7.2 风险分数结构

每帧风险分数为7维向量：

$$
[s_{dl},\ s_{da},\ s_{tl},\ s_{ta},\ dsc,\ tsc,\ total]
$$

| 索引 | 含义           |
| ---- | -------------- |
| 0    | 纵向距离评分   |
| 1    | 横向距离评分   |
| 2    | 纵向TTC评分    |
| 3    | 横向TTC评分    |
| 4    | 距离综合评分   |
| 5    | TTC综合评分    |
| 6    | 总评分（综合） |

### 7.3 评分函数

**距离评分**（0-5分）：

```python
def score_distance(d):
    if abs(d) < 0.3: return 0
    if abs(d) < 0.8: return 1
    if abs(d) < 1.3: return 2
    if abs(d) < 3.0: return 3
    if abs(d) < 5.0: return 4
    return 5
```

**TTC评分**（0-5分）：

```python
def score_ttc(ttc):
    if ttc < 0.15: return 0
    if ttc < 0.5: return 1
    if ttc < 1.0: return 2
    if ttc < 2.0: return 3
    if ttc < 4.0: return 4
    return 5
```

## 8. DFS + Top-K% 危险剪枝算法

### 8.1 伪代码

```
输入: IntentBlocks, K (保留比例), ego_trajectory, target_trajectory
输出: Top-K% 危险轨迹列表

def dfs(block_idx, current_state, trajectory):
    if block_idx == len(IntentBlocks):
        # 到达叶子节点
        variant_trajectory = build_trajectory(trajectory)
        risk_score = compute_risk_with_calculator(variant_trajectory, target_trajectory)
        results.append((variant_trajectory, risk_score))
        return

    block = IntentBlocks[block_idx]
    for mutation in block.mutations:
        # 采样变异值
        delta_a = sample_from_range(mutation.delta_a_range)
        delta_omega = sample_from_range(mutation.delta_omega_range)

        # 计算区块恒定控制量
        a_block = clamp(current_state.a + delta_a, A_MIN, A_MAX)
        omega_block = clamp(current_state.omega + delta_omega, OMEGA_MIN, OMEGA_MAX)

        # 递推该Block内所有帧
        for _ in range(block.frame_count):
            state = kinematically_integrate(current_state, a_block, omega_block)
            trajectory.append(state)
            current_state = state

        # 递归
        dfs(block_idx + 1, current_state, trajectory)

# 主流程
all_variants = []
dfs(0, initial_state, [])

# 按危险分数排序
sorted_variants = sorted(all_variants, key=lambda x: x.risk_score, reverse=True)

# Top-K% 筛选
k = int(len(sorted_variants) * K / 100)
top_k_variants = sorted_variants[:k]
```

### 8.2 复杂度

| 方案   | Block数 | 每Block组合 | 总变体数        | Top-10%后 |
| ------ | ------- | ----------- | --------------- | --------- |
| 旧方案 | 50      | 3           | $3^{50}$      | -         |
| v2方案 | 3~5     | 3           | $3^5 = 243$   | ~24       |
| v3方案 | 3~5     | 9           | $9^5 = 59049$ | ~5900     |
