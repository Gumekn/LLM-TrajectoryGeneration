# 意图驱动轨迹变异穷举方法规范（v3）

## 1. 概述

本规范定义基于驾驶意图的轨迹变异穷举方法。给定一条原始交互轨迹（5s / 50帧 / $\Delta t = 0.1s$），通过意图合并降维、宏动作变异、DFS深度优先搜索和Top-K%危险剪枝，生成所有有效的轨迹变体。

**核心思想**：
- 将高频逐帧变异改为拉长周期的段落式决策（宏动作）
- 加速度变异针对**矢量加速度**（ax, ay合并为标量+方向）
- 保持状态用**随机扰动**而非固定离散值
- 危险分数复用 `processor.py` 的 `RiskCalculator`

### 1.1 数量级爆炸解决方案

| 方案 | 原理 | 效果 |
|------|------|------|
| **意图合并（游程编码）** | 相邻相同意图合并为Intent Block | 8~13个意图帧 → 3~5个Block |
| **加速度矢量变异** | ax, ay合并为矢量，统一变异 | 简化二维变异为一维+方向 |
| **随机扰动保持** | 保持状态时施加小幅随机扰动 | 更真实的驾驶行为 |
| **DFS + Top-K%剪枝** | 生成后按危险分数筛选 | 只保留最危险轨迹 |

## 2. 状态向量定义

### 2.1 运动学状态向量

原始轨迹的每帧数据映射为六维状态向量：

$$S_t = [v_t,\ a_t,\ \theta_t,\ \omega_t,\ x_t,\ y_t]$$

| 符号 | 含义 | 单位 |
|------|------|------|
| $v_t$ | 速度标量 | m/s |
| $a_t$ | 纵向加速度（标量） | m/s^2 |
| $\theta_t$ | 航向角 (heading) | rad |
| $\omega_t$ | 横摆角速度 (yaw rate) | rad/s |
| $x_t$ | X轴坐标 | m |
| $y_t$ | Y轴坐标 | m |

### 2.2 矢量加速度

**原始加速度**：$[a_x,\ a_y]$（二维矢量）

**矢量合并**：
$$a_{mag} = \sqrt{a_x^2 + a_y^2}$$

$$a_{angle} = \text{atan2}(a_y,\ a_x)$$

**局部坐标系映射**：
- $a_x$：纵向加速度（正=加速，负=减速）
- $a_y$：横向加速度（正=左，负=右，向心加速度）

### 2.3 初始状态获取

假设原始轨迹存储于 VehicleTrajectory 结构，时间步长 $\Delta t = 0.1s$：

$$v_0 = |velocities[0]|$$

$$a_{x,0} = accelerations[0][0]$$
$$a_{y,0} = accelerations[0][1]$$
$$a_{mag,0} = \sqrt{a_{x,0}^2 + a_{y,0}^2}$$

$$\theta_0 = headings[0]$$

$$\omega_0 = \frac{headings[1] - headings[0]}{\Delta t}$$

$$x_0 = positions[0][0]$$
$$y_0 = positions[0][1]$$

## 3. 意图合并（游程编码）

### 3.1 问题背景

5秒轨迹中存在8~13个意图帧。若每个意图帧都作为独立变异段，将产生计算爆炸。

### 3.2 游程编码合并算法

**输入**：意图序列

**合并逻辑**：相邻且标签相同的意图合并为一个**意图区块 (Intent Block)**

**输出**：
```
[Block(intent='cruise', start=0.0s, end=0.8s),
 Block(intent='turn_left', start=0.8s, end=1.5s),
 ...]
```

### 3.3 变异触发时机

**只在每个Block的起始帧进行一次变异**：
1. 在Block起始帧根据意图选定变异参数
2. 在该Block持续时间内保持参数恒定（宏动作）
3. 按0.1s步长进行物理积分递推

## 4. 变异变量定义

### 4.1 控制变量边界

| 变量 | 最小值 | 最大值 | 单位 |
|------|--------|--------|------|
| 纵向加速度 $a_x$ | -8.0 | 3.0 | m/s^2 |
| 横向加速度 $a_y$ | -5.0 | 5.0 | m/s^2 |
| 横摆角速度 $\omega$ | -0.35 | 0.35 | rad/s |

### 4.2 变异量定义

**Delta_a（加速度矢量增量）**：
- 针对矢量加速度 $a_{mag}$ 的增量
- 单位：m/s^2
- 含义：改变当前加速度的大小

**Delta_omega（横摆角速度增量）**：
- 针对横摆角速度 $\omega$ 的增量
- 单位：rad/s
- 含义：改变转向角速度

### 4.3 随机扰动规则

当意图要求"保持当前状态"时，使用**随机扰动**而非固定值：

$$Delta\_a \sim U(-\sigma_a,\ \sigma_a)$$

$$Delta\_omega \sim U(-\sigma_\omega,\ \sigma_\omega)$$

| 扰动参数 | 值 |
|----------|-----|
| $\sigma_a$（加速度标准差） | 0.05 m/s^2 |
| $\sigma_\omega$（角速度标准差） | 0.004 rad/s (约0.23 deg/s) |

## 6. 宏动作递推公式

### 6.1 区块恒定控制量计算

在Block起始帧，根据变异方案计算控制量：

**加速度更新**：
$$a_{block} = \text{clamp}(a_{prev} + \Delta a,\ a_{min},\ a_{max})$$

**横摆角速度更新**：
$$\omega_{block} = \text{clamp}(\omega_{prev} + \Delta \omega,\ \omega_{min},\ \omega_{max})$$

### 6.2 微步长递推公式

在Block内的每一帧，执行0.1s的物理递推：

**速度更新**：
$$v_t = \max(0,\ v_{t-1} + a_{block} \cdot 0.1)$$

**航向角更新**：
$$\theta_t = \theta_{t-1} + \omega_{block} \cdot 0.1$$

**位置更新（运动学积分）**：
$$x_t = x_{t-1} + v_t \cdot \cos(\theta_t) \cdot 0.1$$

$$y_t = y_{t-1} + v_t \cdot \sin(\theta_t) \cdot 0.1$$

### 6.3 VehicleTrajectory 数据结构映射

| 字段 | 映射 |
|------|------|
| `positions[t]` | $[x_t,\ y_t,\ z_{t-1}]$ |
| `headings[t]` | $\theta_t$ |
| `velocities[t]` | $[v_t,\ 0]$ |
| `accelerations[t]` | $[a_{block},\ v_t \cdot \omega_{block}]$ |
| `valid[t]` | True |

## 7. 危险分数计算（复用processor.py）

### 7.1 RiskCalculator复用

使用与 `processor.py` 相同的 `RiskCalculator` 计算危险分数：

```python
from core.processor import RiskCalculator, VehicleTrajectory

# 构建轨迹对象
ego_traj = VehicleTrajectory(...)
target_traj = VehicleTrajectory(...)

# 计算风险
calculator = RiskCalculator(ego_trajectory, target_trajectory)
calculator.compute_relative_metrics()
calculator.compute_risk_scores()

# 获取危险分数
risk_scores = calculator.risk_scores  # (N, 7)
max_risk = np.min(risk_scores[:, 6])  # 取第7列（综合评分）最小值
```

### 7.2 风险分数结构

每帧风险分数为7维向量：
$$[s_{dl},\ s_{da},\ s_{tl},\ s_{ta},\ dsc,\ tsc,\ total]$$

| 索引 | 含义 |
|------|------|
| 0 | 纵向距离评分 |
| 1 | 横向距离评分 |
| 2 | 纵向TTC评分 |
| 3 | 横向TTC评分 |
| 4 | 距离综合评分 |
| 5 | TTC综合评分 |
| 6 | 总评分（综合） |

### 7.3 评分函数

**距离评分**（0-5分）：
```python
def score_distance(d):
    if abs(d) < 0.3: return 0
    if abs(d) < 0.8: return 1
    if abs(d) < 1.3: return 2
    if abs(d) < 3.0: return 3
    if abs(d) < 5.0: return 4
    return 5
```

**TTC评分**（0-5分）：
```python
def score_ttc(ttc):
    if ttc < 0.15: return 0
    if ttc < 0.5: return 1
    if ttc < 1.0: return 2
    if ttc < 2.0: return 3
    if ttc < 4.0: return 4
    return 5
```

## 8. DFS + Top-K% 危险剪枝算法

### 8.1 伪代码

```
输入: IntentBlocks, K (保留比例), ego_trajectory, target_trajectory
输出: Top-K% 危险轨迹列表

def dfs(block_idx, current_state, trajectory):
    if block_idx == len(IntentBlocks):
        # 到达叶子节点
        variant_trajectory = build_trajectory(trajectory)
        risk_score = compute_risk_with_calculator(variant_trajectory, target_trajectory)
        results.append((variant_trajectory, risk_score))
        return

    block = IntentBlocks[block_idx]
    for mutation in block.mutations:
        # 采样变异值
        delta_a = sample_from_range(mutation.delta_a_range)
        delta_omega = sample_from_range(mutation.delta_omega_range)

        # 计算区块恒定控制量
        a_block = clamp(current_state.a + delta_a, A_MIN, A_MAX)
        omega_block = clamp(current_state.omega + delta_omega, OMEGA_MIN, OMEGA_MAX)

        # 递推该Block内所有帧
        for _ in range(block.frame_count):
            state = kinematically_integrate(current_state, a_block, omega_block)
            trajectory.append(state)
            current_state = state

        # 递归
        dfs(block_idx + 1, current_state, trajectory)

# 主流程
all_variants = []
dfs(0, initial_state, [])

# 按危险分数排序
sorted_variants = sorted(all_variants, key=lambda x: x.risk_score, reverse=True)

# Top-K% 筛选
k = int(len(sorted_variants) * K / 100)
top_k_variants = sorted_variants[:k]
```

### 8.2 复杂度

| 方案 | Block数 | 每Block分支 | 总变体数 | Top-K%后 |
|------|---------|-------------|----------|----------|
| 旧方案 | 50 | 3 | $3^{50}$ | - |
| v2方案 | 3~5 | 3 | $3^5 = 243$ | ~24 |
| v3方案 | 3~5 | 3~9 | <500 | ~50 |
